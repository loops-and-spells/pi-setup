# pi-setup

## Purpose

This repo is a **suite for standing up a complete pi-based harness** — everything a machine needs to run [pi](https://pi.dev) the way the owner works: local inference engines (managed by the `pi-engine` CLI), the drop-in agent harness (global DOX contract, subagents, prompts, extensions, package list, installed by `scripts/install-harness.sh`), and the benchmarking that justifies the engine choices.

The targeted outcome, which every change must serve:

- **Drop-in**: a fresh machine goes from clone to working pi harness with `bun install`, `bun run harness:install`, `bun run engine install`, `bun run engine use <engine>` — nothing more
- **Repo as source of truth**: user-authored pi config lives here and is symlinked out; never grow setup steps that require hand-editing files under `~/.pi/agent`
- **No hidden machine coupling**: machine-specific values belong in generated files (`models.json`, systemd units) or documented env tunables, not hardcoded in the harness payload
- **Idempotent by contract**: every installer/setup path must be safely re-runnable

## Ownership

- `apps/pi-engine` — Effect-based CLI: engine lifecycle (llama/vllm/ds4/council), systemd units, writes `~/.pi/agent/models.json` and settings defaults
- `apps/pi-council` — pi extension (symlinked into `~/.pi/agent/extensions/pi-council.ts`); owns `~/.pi/agent/council.json` at runtime
- `apps/pi-bench` — engine/model benchmarking
- `packages/core` — shared paths and pi config helpers (`paths.ts`, `pi-config.ts`)
- `harness/` — source of truth for everything symlinked into `~/.pi/agent`; see its AGENTS.md
- `scripts/install-harness.sh` — idempotent drop-in installer (`bun run harness:install`)
- `legacy/` — retired bash scripts, kept for reference only; do not extend

## Local Contracts

- Only one engine runs at a time (VRAM constraint); engine switching must go through `pi-engine use`, never raw systemctl/nohup
- `~/.pi/agent/models.json` is generated by `pi-engine install` — never hand-edit or vendor it
- Machine-local runtime state (`settings.json`, `council.json`, sessions, auth) is never committed; the installer merges, it does not overwrite
- Claude Code does not read AGENTS.md natively: every AGENTS.md in this repo has a sibling `CLAUDE.md` containing only `@AGENTS.md`; add the shim whenever a child AGENTS.md is created

## Verification

- `bun run typecheck` and `bun run test` (turbo)
- `bash scripts/install-harness.sh` must be re-runnable with no changes on a converged machine
